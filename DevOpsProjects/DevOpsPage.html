Project Overview

The eMarketing Corporation is developing a comprehensive web application enabling business customers to order and monitor services efficiently. The development team structure includes:

  • 4 dedicated developers
  • 1 UX designer
  • 1 product owner
  • Total of 22 developers across 5 teams

The tech stack primarily consists of Java and Microsoft SQL Server.

Existing Bad Practices and Improvements

During the initial assessment of the eMarketing Corporation's operations, several bad practices were identified:

  • Lack of Automation: Many deployment processes were manual, leading to errors and inconsistencies.
  • Poor Backlog Management: The backlog prioritization was often neglected, resulting in confusion among team members.
  • Inefficient Resource Management: The infrastructure was not optimized, leading to increased operational costs.
  • Slow Deployment Times: The deployment frequency was low due to cumbersome processes, affecting overall productivity.
  • Limited Rollback Capabilities: Rollback procedures were complex and time-consuming, risking service availability during failures.

Improvements Implemented

To address these challenges, the following improvements were made:

  • Implemented automated deployment pipelines to streamline the deployment process.
  • Established a structured backlog management system to improve prioritization and task clarity.
  • Optimized infrastructure and resource allocation to reduce operational costs.
  • Adopted a Blue-Green deployment strategy to enhance deployment frequency and minimize downtime.
  • Developed simplified rollback procedures, allowing for quicker responses to issues.

Proposed Solutions

Existing Environment

  • Development tasks in backlog kept on sticky notes, no capacity/sprint planning in place.
  • Current development environment runs ad-hoc local versioning from individual workstations on-site – No source control in place.
  • Package building done ad-hoc, by an individual – Requires replacement.
  • Existing Production Environment Resources configured for BLUE/GREEN Deployment.

Proposed Solution - Plan/Code

  • Use Azure Boards for backlog management.
  • Utilize Eclipse IDE for development in the Green environment.
  • Implement Git Branches for version control.
  • Connect Eclipse to Git Repository for secure storage.

Proposed Solution - Build/Test

  • Use Eclipse IDE compiler for automated testing.
  • Implement Jenkins for CI/CD to build and push application packages.
  • Conduct Regression or Integrated Testing in the Green environment.

Proposed Solution - Release/Deploy

  • Route production traffic from Blue to Green environment.
  • Quickly roll back to the Blue environment if issues arise.
  • Ensure fast recovery and meeting business requirements.

Proposed Solution - Operate/Monitor

  • Implement Ansible for configuration management and monitoring.
  • Ensure agile and responsive control over production software.
